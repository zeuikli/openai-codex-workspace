# ChatGPT 快速 Pilot

新 thread 或 agent 可選模型時，使用 `gpt-5.6-luna` 搭配 `medium` reasoning；否則只套用 `cost` 工作流，不宣稱已切換目前模型。

## 前置條件

- 確認任務範圍小、可回復且能直接驗證。
- 找出精確檔案與 repo 內的正向參考模式。
- 說明需求詮釋、關鍵假設與可執行的 Done when。
- 任務擴大成架構、安全或跨模組推理時停止，改用更高強度 profile。

## 工作流

1. 只讀目標、直接 caller、相關測試與目前 diff。
2. 優先沿用既有模式，不新增未被證明必要的抽象。
3. 實作最小且維持既有行為的修改。
4. 執行最關鍵的測試，再跑一項結構檢查，例如 `rg` 或 `git diff --check`。
5. 明列未檢查範圍，不暗示已完成全面驗證。

## 來源紀律

- 以使用者提供來源或 primary source 驗證事實。
- Repo 相關宣稱附上檔案與行號。
- 無法確認的事實標為未驗證，不虛構數字或引用。

## 停止條件

- 兩輪修正仍失敗時停止，診斷缺少的 context。
- 不用擴大 scope 補償需求歧義。
- 使用者未明確要求 agent 或平行工作時不委派。

## 輸出

依序回覆：結論、修改檔案、驗證結果、未完成風險。
